================================================================================
THE GRAMMAR
================================================================================

Program → Stmt
Stmt → Expr ';' | Cond ';'
Expr → Factor Expr'
Expr' → '+' Factor Expr' | ε
Factor → id | call | '(' Expr ')'
call → id '(' ')'
Cond → Expr '==' Expr | '(' Cond ')'

================================================================================
COMPUTED SETS
================================================================================

NULLABLE:
  Expr'

FIRST:
  Program: {id, '('}
  Stmt:    {id, '('}
  Expr:    {id, '('}
  Expr':   {'+', ε}
  Factor:  {id, '('}
  call:    {id}
  Cond:    {id, '('}

FOLLOW:
  Program: {$}
  Stmt:    {$}
  Expr:    {';', '==', ')'}
  Expr':   {';', '==', ')'}
  Factor:  {'+', ';', '==', ')'}
  call:    {'+', ';', '==', ')'}
  Cond:    {';', ')'}

================================================================================
CONFLICTS
================================================================================

1. Stmt on 'id':
   - Stmt → Expr ';'
   - Stmt → Cond ';'
   Reason: Both Expr and Cond can start with 'id'

2. Stmt on '(':
   - Stmt → Expr ';'
   - Stmt → Cond ';'
   Reason: Both Expr and Cond can start with '('

3. Factor on 'id':
   - Factor → id
   - Factor → call
   Reason: Both productions start with 'id'

4. Cond on '(':
   - Cond → Expr '==' Expr
   - Cond → '(' Cond ')'
   Reason: Expr can start with '(', creating ambiguity

================================================================================
KEY LEARNING POINTS
================================================================================

1. NULLABLE Computation:
   - Only Expr' is nullable (has ε production)
   - Important for FOLLOW computation

2. FIRST Sets:
   - Most non-terminals have FIRST = {id, '('}
   - This similarity causes conflicts

3. FOLLOW Sets:
   - Computed by looking at what comes after each non-terminal
   - Used when ε is in FIRST

4. Parsing Table:
   - Built using FIRST and FOLLOW
   - Conflicts occur when multiple entries in same cell

5. LL(1) Determination:
   - Grammar is LL(1) if no conflicts
   - This grammar has 4 conflicts → NOT LL(1)

================================================================================
COMMON MISTAKES TO AVOID
================================================================================

1. ❌ Forgetting to add $ to FOLLOW of start symbol
2. ❌ Not handling nullable non-terminals in FIRST computation
3. ❌ Missing ε productions when building table
4. ❌ Not using FOLLOW when ε is in FIRST
5. ❌ Confusing FIRST and FOLLOW

PRACTICE TIPS

2. Double-check:

3. Mark conflicts clearly:

   - Explain reasoning
   - Make it easy to follow
